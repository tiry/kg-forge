Storing AI Enabled Content

We want to store into the Content Lake:

  • the content we receive via Ingestion

    • Blobs

    • Meta-data

    • ACLs

  • the Curated Content we extract from Ingested documents

    • Text Chunks

    • Embeddings associated to Chunks

  • new meta-data generated by Context

    • Meta-data

    • Named Entities

    • Relations

All these data need to be stored safely and with respect for security - thanks to support for complex types and complex schema we can configure HXPR to store all that:

  • Blobs and large texts can be stored in S3

  • Common Meta-data store

  • Extended meta-data as JSON ?

  • Entities as Directory entries

  • Relations as Document?

  • Embeddings as a complex array containing vectors and pointers to Chunks

Indexing

We can rely on the scalable and pluggable indexing infrastructure of HXPR to:

  • do the default fulltext / keyword indexing

  • create a vector index into OpenSearch

  • Store denormalized records on S3 (indexing via Athena or Redshift)

This “indexed” data is considered as derived data that can be flushed / rebuilt at any point in time.
